## 📝 Part 1. 강의 내용 정리

### 1. Java 구동 원리
자바는 환경에 구애받지 않는 실행 구조를 가지고 있는 것이 가장 큰 장점입니다.

| 단계 | 과정 | 결과물 | 비고 |
| :--- | :--- | :--- | :--- |
| **1. 작성** | 소스 코드 작성 | `.java` 파일 | 개발자가 읽는 코드 |
| **2. 컴파일** | 자바 컴파일러(`javac`) 사용 | `.class` 파일 | 바이트코드 (컴파일 단계) |
| **3. 실행** | **JVM** 호출 | 프로그램 실행 | 운영체제에 독립적 (**WORA**) |

<br>

### 2. 변수 (Variable)
변수는 데이터를 보관했다가 필요할 때 꺼내 쓰는 개념입니다.

**[주요 타입]**
| 타입 | 종류 | 크기/특징 |
| :--- | :--- | :--- |
| `int` | 정수 | 정수 기본형 (예: 1, 100) |
| `double` | 실수 | 실수 기본형 (예: 1.5, 10.0) |
| `boolean` | 불리언 | `true`, `false`만 가능 |
| `char` | 문자 | 2바이트, 문자 하나(`'A'`) |
| `String` | 문자열 | 문장(`"Hello"`) - **참조 타입** |

> ⚠️ **변수 초기화 오류 (`variable a might not have been initialized`)**
> 메모리는 여러 시스템이 공유하는 공간입니다. 이전에 사용하던 **쓰레기 값(Garbage Value)**이 남아있을 수 있기 때문에, 자바는 안전을 위해 초기화를 강제하고 이를 어기면 컴파일 에러를 냅니다.

> ✅ **몰랐던 용어: 리터럴(Literal)**
> 개발자가 코드에 직접 적은 고정된 값 (예: `10`, `3.14`, `"Java"`)을 의미합니다.

<br>

### 3. 자바의 명명 규칙 (Naming Convention)
가독성과 협업을 위해 자바에서 약속한 규칙입니다.

* **클래스 (PascalCase):** 첫 글자 대문자, 나머지는 낙타 표기법 (예: `UserMain`)
* **그 외 나머지 (camelCase):** 변수, 메서드 등은 첫 글자 소문자 시작 (예: `userName`)
* **상수 (UPPER_SNAKE_CASE):** 모두 대문자 + 언더바 (예: `MAX_VALUE`)
* **패키지:** 모두 소문자 (예: `variable.ex`)

<br>

### 4. 연산자 (Operator)
연산의 우선순위와 증감 연산자의 위치가 중요합니다.

* **증감 연산자**
  * `++a` (Prefix): 값을 먼저 증가시킨 후 연산에 참여
  * `a++` (Postfix): 연산을 먼저 수행한 후 값을 증가
* **문자열 비교:** `==`가 아닌 **`.equals()`** 메서드를 사용해야 정확한 비교가 가능합니다.

<br>

### 5. 조건문 (Conditionals)
* **Switch의 변화**
  * **기본 특징:** `break`가 없으면 아래 조건이 연달아 실행되는 **Fall-through** 현상이 발생합니다.
  * **Java 14+ 새로운 Switch:** 화살표(`->`)를 사용하며, 별도의 `break` 없이도 깔끔하게 작성이 가능해졌습니다.

```java
// 새로운 switch 예시
switch (grade) {
    case "A" -> System.out.println("탁월");
    case "B" -> System.out.println("우수");
    default -> System.out.println("보통");
}
```

### 6. 반복문 (Loop)
| 종류 | 특징 | 주요 사용 상황 |
| :--- | :--- | :--- |
| **while** | 조건을 먼저 확인하고 실행 | 반복 횟수를 정확히 모를 때 (예: 특정 값이 들어올 때까지) |
| **do-while** | **무조건 한 번은 실행** 후 조건 확인 | 사용자 입력 등을 받고나서 검사해야 할 때 |
| **for** | 초기식, 조건식, 증감식이 한 세트 | **반복 횟수가 명확할 때** (배열 순회 등) |
