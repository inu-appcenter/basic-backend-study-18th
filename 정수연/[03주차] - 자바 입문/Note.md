# 지역변수
* *특정 지역에서만(={})* 사용할 수 있는 변수


* 자신의 코드 블록을 벗어나면 제거되기 때문에 접근할 수 없다.

# 스코프(Scope)
* 변수의 접근 가능한 범위


* for문의 경우도 for문 안에서 선언한 변수는 {} 안에서만 사용할 수 있다.


* 메모리 사용과 유지보수 관점에서 for문이 while문 보다 좋음


# 형변환
- 작은 범위에서 큰 범위로는 자동으로 형 변환이 되어 값을 넣을 수 있지만 큰 범위에서 작은 범위는 문제가 발생함


- 따라서 큰 범위에서 작은 범위 대입은 **명시적 형변환**이 필요하다

### 명시적 형변환
* `intValue = (int) doubleValue;`

### 오버플로우
* 숫자의 기존 범위를 초과
1. 같은 타입끼리의 계산은 같은 타입의 결과를 낸다.


2. 서로 다른 타입의 계산은 큰 범위로 자동 형변환이 일어난다.

# 입력받기
* `Scanner scanner = new Scanner(System.in);`

*`Scanner scanner` : scanner 변수를 선언

* `System.in` : 사용자 입력 받기


* `scanner.nextLine()` : 엔터(\n)을 입력할 때까지 문자를 가져온다.


* `scanner.nextInt()` : 입력을 int형으로 가져온다.


* `scanner.nextDouble()` : 입력을 double형으로 가져온다.

### 참조값
* 생성한 배열을 참조할 수 있는 메모리의 주소를 나타내는 특별한 값

# 배열
- 배열은 **0부터** 시작


- 접근 가능한 배열의 인덱스 범위를 벗어나면 오류 발생


- 기본형 : 변수에 사용할 값을 직접 넣을 수 있는 데이터 타입
  ex) `int`, `long`, `double`, `boolean`


- 참조형 : 데이터에 접근하기 위한 참조(주소)를 저장하는 데이터 타입
  ex) `int[] students`


- 동적 메모리 할당 : 크기를 동적으로 할당하여 크기가 변해서 유연성을 제공할 수 있다.


### 2차원 배열
* `arr[row][column]`

* `arr.length` : 헹의 길이

* `arr[row].length` : 열의 길이

# 메서드
- 메서드 선언 : `public static int add(int a, int b)`


- 함수에 값을 입력하면 어떤 연산을 처리한 다음에 결과를 변환한다.


---

- 메서드 호출할 때에는 **메서드에 넘기는 값과 매개변수(파라미터)의 타입**이 맞아야한다.


- 매개변수 : 메서드 호출부와 메서드 내부 사이에서 값을 전달하는 역할을 하는 변수


- 메서드가 값을 반환하지 않는 경우 `void` 사용

## 자바는 항상 변수의 값을 복사해서 대입한다.
