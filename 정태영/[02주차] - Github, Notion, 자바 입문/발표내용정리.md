### 메서드명

---

동사 사용

| **동사** | **사용 용도** | **예시** |
| --- | --- | --- |
| **get** | 데이터 반환 | `getNumber(){}` |
| set | 데이터 주입/설정 | `setNumber(int number){}` |
| **init** | 데이터를 초기화 | `void initDate(){}` |
| **is/has** | 참/거짓, 데이터 유무 확인 | `boolean isEmpty(){}` 
`boolean hasNextToken(){}`  |
| **create** | 새로운 객체 생성 | `Node createNode(){}` |
| **delete/remove** | 객체 삭제 | `Node removeNode(){}` |
| **find** | 데이터를 찾기 | `Node findNode(int index){}` |
| **to** | 다른 형태의 객체로 변환 | `String toString(){}` |
| **A-By-B** | B를 기준으로 A를  | `void getUserById(String id){}` |

## 클린 코드

![image.png](image.png)

**효율적이고 가독성 높은 클린 코드를 작성하는 원칙을 소개하는 책이다.**

### 1. 의도를 분명히 밝혀라

---

변수 이름만 보고 이 변수가 **왜 존재하는지**, **무엇을 하는지**, **어떻게 쓰이는지** 알 수 있어야 함

- **Bad** ❌
    
    `int d; // 경과 시간(일 단위)`
    
- **Good** ✅
    
    `int elapsedTimeInDays;`
    

### 2. 의미 있게 구분하라

---

단순히 숫자를 붙여서 구분 ❌ → 차이가 명확하게

- **Bad** ❌
    
    `copyChars(char[] a1, char[] a2)`
    
- **Good** ✅
    
    `copyChars(char[] source, char[] destination)`
    

### 3. 발음하기와 검색하기 쉬운 이름을 사용하라

---

- **Bad** ❌
    
    `long genymdhms;`
    
- **Good** ✅
    
    `long generationTimestamp;` 
    

### 4. 한 개념에 한 단어를 사용하라

---

개념 하나에 단어 하나를 사용하고 일관성 있게 사용

- **Bad** ❌
    
    `DeviceManager`에는 `fetch` 
    
    `UserManager`에는 `get`
    
    `PostManager`에는 `retrieve`를 사용
    
- **Good** ✅
    
    `get`으로 통일
